# /sc:test - 테스트 및 품질 보증

## 트리거 (Triggers)
- 단위, 통합 또는 e2e 테스트에 대한 테스트 실행 요청
- 커버리지 분석 및 품질 게이트 검증 필요
- 지속적인 테스트 및 감시 모드 시나리오
- 테스트 실패 분석 및 디버깅 요구사항

## 사용법 (Usage)
```
/sc:test [target] [--type unit|integration|e2e|all] [--coverage] [--watch] [--fix]
```

## 행동 흐름 (Behavioral Flow)
1. **탐색 (Discover)**: 러너 패턴 및 규칙을 사용하여 사용 가능한 테스트 분류
2. **구성 (Configure)**: 적절한 테스트 환경 및 실행 매개변수 설정
3. **실행 (Execute)**: 모니터링 및 실시간 진행 상황 추적을 통해 테스트 실행
4. **분석 (Analyze)**: 커버리지 보고서 및 실패 진단 생성
5. **보고 (Report)**: 실행 가능한 추천 사항 및 품질 메트릭 제공

주요 행동:
- 테스트 프레임워크 및 구성 자동 감지
- 메트릭이 포함된 포괄적인 커버리지 보고서 생성
- e2e 브라우저 테스트를 위해 Playwright MCP 활성화
- 지능적인 테스트 실패 분석 제공
- 개발 중 지속적인 감시 모드 지원

## MCP 통합 (MCP Integration)
- **Playwright MCP**: `--type e2e` 브라우저 테스트를 위해 자동 활성화
- **QA 전문가 페르소나**: 테스트 분석 및 품질 평가를 위해 활성화
- **향상된 기능**: 교차 브라우저 테스트, 시각적 검증, 성능 메트릭

## 도구 협력 (Tool Coordination)
- **Bash**: 테스트 러너 실행 및 환경 관리
- **Glob**: 테스트 탐색 및 파일 패턴 매칭
- **Grep**: 결과 파싱 및 실패 분석
- **Write**: 커버리지 보고서 및 테스트 요약

## 주요 패턴 (Key Patterns)
- **테스트 탐색 (Test Discovery)**: 패턴 기반 분류 → 적절한 러너 선택
- **커버리지 분석 (Coverage Analysis)**: 실행 메트릭 → 포괄적인 커버리지 보고
- **E2E 테스트 (E2E Testing)**: 브라우저 자동화 → 교차 플랫폼 검증
- **감시 모드 (Watch Mode)**: 파일 모니터링 → 지속적인 테스트 실행

## 예시 (Examples)

### 기본 테스트 실행
```
/sc:test
# 표준 구성으로 모든 테스트를 탐색하고 실행
# 통과/실패 요약 및 기본 커버리지 생성
```

### 대상 커버리지 분석
```
/sc:test src/components --type unit --coverage
# 상세 커버리지 메트릭으로 특정 디렉토리에 대한 단위 테스트
```

### 브라우저 테스트
```
/sc:test --type e2e
# 포괄적인 브라우저 테스트를 위해 Playwright MCP 활성화
# 교차 브라우저 호환성 및 시각적 검증
```

### 개발 감시 모드
```
/sc:test --watch --fix
# 자동 간단한 실패 수정을 통한 지속적인 테스트
# 개발 중 실시간 피드백
```

## 경계 (Boundaries)

**수행할 작업:**
- 프로젝트의 구성된 테스트 러너를 사용하여 기존 테스트 스위트 실행
- 커버리지 보고서 및 품질 메트릭 생성
- 실행 가능한 추천 사항과 함께 지능적인 테스트 실패 분석 제공

**수행하지 않을 작업:**
- 테스트 케이스 생성 또는 테스트 프레임워크 구성 수정
- 적절한 설정 없이 외부 서비스가 필요한 테스트 실행
- 명시적인 허가 없이 테스트 파일을 파괴적으로 변경
