# /sc:build - 프로젝트 빌드 및 패키징

## 트리거 (Triggers)
- 다양한 환경에 대한 프로젝트 컴파일 및 패키징 요청
- 빌드 최적화 및 아티팩트 생성 필요
- 빌드 과정 중 오류 디버깅
- 배포 준비 및 아티팩트 패키징 요구사항

## 사용법 (Usage)
```
/sc:build [target] [--type dev|prod|test] [--clean] [--optimize] [--verbose]
```

## 행동 흐름 (Behavioral Flow)
1. **분석 (Analyze)**: 프로젝트 구조, 빌드 구성 및 종속성 매니페스트 분석
2. **검증 (Validate)**: 빌드 환경, 종속성 및 필수 툴체인 컴포넌트 검증
3. **실행 (Execute)**: 실시간 모니터링 및 오류 감지를 통한 빌드 프로세스 실행
4. **최적화 (Optimize)**: 빌드 아티팩트 최적화, 최적화 적용 및 번들 크기 최소화
5. **패키지 (Package)**: 배포 아티팩트 및 포괄적인 빌드 보고서 생성

주요 행동:
- 종속성 검증을 통한 구성 기반 빌드 오케스트레이션
- 실행 가능한 해결 가이드를 통한 지능적인 오류 분석
- 환경별 최적화 (개발/프로덕션/테스트 구성)
- 타이밍 메트릭 및 아티팩트 분석을 포함한 포괄적인 빌드 보고

## MCP 통합 (MCP Integration)
- **Playwright MCP**: 빌드 중 빌드 검증 및 UI 테스트를 위해 자동 활성화
- **DevOps 엔지니어 페르소나**: 빌드 최적화 및 배포 준비를 위해 활성화
- **향상된 기능**: 빌드 파이프라인 통합, 성능 모니터링, 아티팩트 검증

## 도구 협력 (Tool Coordination)
- **Bash**: 빌드 시스템 실행 및 프로세스 관리
- **Read**: 구성 분석 및 매니페스트 검사
- **Grep**: 오류 파싱 및 빌드 로그 분석
- **Glob**: 아티팩트 탐색 및 검증
- **Write**: 빌드 보고서 및 배포 문서화

## 주요 패턴 (Key Patterns)
- **환경 빌드 (Environment Builds)**: 개발/프로덕션/테스트 → 적절한 구성 및 최적화
- **오류 분석 (Error Analysis)**: 빌드 실패 → 진단 분석 및 해결 가이드
- **최적화 (Optimization)**: 아티팩트 분석 → 크기 축소 및 성능 향상
- **검증 (Validation)**: 빌드 검증 → 품질 게이트 및 배포 준비 상태

## 예시 (Examples)

### 표준 프로젝트 빌드
```
/sc:build
# 기본 구성을 사용하여 전체 프로젝트 빌드
# 아티팩트 및 포괄적인 빌드 보고서 생성
```

### 프로덕션 최적화 빌드
```
/sc:build --type prod --clean --optimize
# 고급 최적화가 적용된 클린 프로덕션 빌드
# 최소화, 트리 쉐이킹 및 배포 준비
```

### 특정 컴포넌트 빌드
```
/sc:build frontend --verbose
# 상세 출력을 통해 특정 프로젝트 컴포넌트 빌드
# 실시간 진행 상황 모니터링 및 진단 정보
```

### 검증을 포함한 개발 빌드
```
/sc:build --type dev --validate
# Playwright 검증을 포함한 개발 빌드
# UI 테스트 및 빌드 검증 통합
```

## 경계 (Boundaries)

**수행할 작업:**
- 기존 구성을 사용하여 프로젝트 빌드 시스템 실행
- 포괄적인 오류 분석 및 최적화 추천 사항 제공
- 상세 보고와 함께 배포 준비가 된 아티팩트 생성

**수행하지 않을 작업:**
- 빌드 시스템 구성 수정 또는 새 빌드 스크립트 생성
- 누락된 빌드 종속성 또는 개발 도구 설치
- 아티팩트 준비를 넘어선 배포 작업 실행
