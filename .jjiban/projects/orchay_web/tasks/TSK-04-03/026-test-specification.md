### 4.2 매뉴얼 테스트 상세

#### TC-001: Web UI 섹션 존재

**테스트 목적**: README.md에 Web UI 관련 섹션이 추가되었는지 확인

**테스트 단계**:
1. `orchay/README.md` 파일 열기
2. "Web UI" 또는 관련 섹션 검색
3. 섹션 존재 여부 확인

**예상 결과**:
- "Web UI" 섹션이 존재함
- 섹션 제목이 명확함

**검증 기준**:
- [ ] Web UI 섹션 존재
- [ ] 섹션 위치가 적절함 (CLI Usage 근처)

---

#### TC-002: CLI 옵션 완전성

**테스트 목적**: 모든 웹 관련 CLI 옵션이 문서화되었는지 확인

**테스트 단계**:
1. Web UI 섹션에서 CLI 옵션 목록 확인
2. 다음 옵션들이 모두 문서화되었는지 확인:
   - `--web`
   - `--web-only`
   - `--port`
   - `--no-web`

**예상 결과**:
- 4개 옵션 모두 문서화됨
- 각 옵션에 대한 설명 존재
- 기본값 명시됨

**검증 기준**:
- [ ] `--web` 옵션 문서화
- [ ] `--web-only` 옵션 문서화
- [ ] `--port` 옵션 문서화 (기본값 8080)
- [ ] `--no-web` 옵션 문서화

---

#### TC-003: 스크린샷 존재 (선택)

**테스트 목적**: 웹 UI 스크린샷이 포함되었는지 확인

**테스트 단계**:
1. Web UI 섹션에서 이미지 참조 확인
2. 이미지 파일 존재 여부 확인
3. 이미지 렌더링 확인

**예상 결과**:
- 스크린샷 이미지가 존재하거나 없음 (선택 사항)

**검증 기준**:
- [ ] 스크린샷 포함됨 (선택)

---

#### TC-004: 사용 예시 정확성

**테스트 목적**: 문서의 사용 예시가 실제 실행 가능한지 확인

**테스트 단계**:
1. Web UI 섹션의 코드 블록 확인
2. 명령어 형식 검토
3. 선택적으로 실제 실행 테스트

**예상 결과**:
- 코드 블록이 bash 형식으로 작성됨
- 명령어가 실제 CLI와 일치함

**검증 기준**:
- [ ] `orchay PROJECT --web` 형식 사용
- [ ] 명령어가 실행 가능함

---

#### TC-005: 기본 포트 번호

**테스트 목적**: 기본 포트 번호가 정확히 명시되었는지 확인

**테스트 단계**:
1. `--port` 옵션 설명 확인
2. 기본값이 8080으로 명시되었는지 확인
3. 접속 URL 예시 확인

**예상 결과**:
- 기본 포트 8080 명시
- `http://localhost:8080` 접속 안내

**검증 기준**:
- [ ] 기본 포트 8080 명시됨
- [ ] 접속 URL 안내 포함
